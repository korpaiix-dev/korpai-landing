## TL;DR (อ่าน 60 วินาที — คำตอบสั้น)

**AI chatbot ฟรี "มีจริง" แต่ "ฟรีไม่จริง"** — ทุกตัวเลือกฟรีในปี 2026 มีต้นทุนแฝง 4 ก้อนเสมอ: (1) **ค่าข้อความเกินโควตา** เช่น LINE OA แผนฟรี broadcast ได้ราว 300 ข้อความ/เดือน เกินต้องขยับแพ็กเกจ ~1,200 บาท/เดือน, (2) **ค่า token LLM** ภาษาไทยกินมากกว่าอังกฤษ 2–3 เท่า, (3) **ค่าเวลาตัวเอง** ติดตั้ง 10–20 ชม. + ดูแล 2–5 ชม./สัปดาห์, (4) **ค่าความเสี่ยง** ตอบผิด/หลอนโดยไม่มี guardrail. กฎง่าย: **ลูกค้าทักน้อยกว่า ~100 แชต/เดือน + คำถามซ้ำๆ → ของฟรีเอาอยู่.** เกินกว่านั้น หรือมีเรื่องจอง/สต็อก/ราคาแพง → ต้นทุนแฝงของฟรีจะแพงกว่าจ่ายจริง.

## "ฟรี" ในตลาด AI chatbot 2026 หมายถึงอะไรกันแน่

คำว่า AI chatbot ฟรี ในปี 2026 มี 3 ความหมายที่คนละเรื่องกันเลย และคนเริ่มต้นมักสับสน:

**แบบที่ 1: ฟีเจอร์ฟรีของแพลตฟอร์มแชต** — LINE OA แผนฟรีมี auto-reply ตามคีย์เวิร์ด, Meta (Messenger/IG) มี saved replies และ automation พื้นฐานฟรี. ข้อจำกัดคือมันเป็น "ตอบตามสคริปต์" ไม่ใช่ AI จริง — พอลูกค้าพิมพ์นอกคีย์เวิร์ด บอตก็เงียบ.

**แบบที่ 2: Free tier ของแพลตฟอร์ม chatbot** — ZWIZ.AI, Botnoi (เจ้าไทย), Tidio, Manychat (เจ้าสากล) ให้ใช้ฟรีแบบจำกัดจำนวนแชต/contact/ฟีเจอร์ เพื่อหวังให้เราอัปเกรด. ของจริงใช้ได้ แต่เพดานต่ำและมักติดโลโก้ "powered by" ของเขา.

**แบบที่ 3: Open source ที่ต้อง self-host** — Dify, Flowise, Typebot, Chatwoot, n8n ตัวซอฟต์แวร์ฟรี 100% แต่เราต้องจ่ายค่า VPS (~250–500 บาท/เดือน) + ค่า API LLM + เวลาตัวเองในการตั้งและดูแลระบบ.

ทั้ง 3 แบบ "เริ่มได้ที่ 0 บาท" จริง — แต่ไม่มีแบบไหน "รันต่อเนื่องที่ 0 บาท" ได้เลยถ้าธุรกิจมีลูกค้าจริง. ส่วนถัดไปคือสมการต้นทุนแฝงที่เราใช้คำนวณให้ลูกค้าก่อนตัดสินใจทุกครั้ง.

## ต้นทุนแฝง 4 ก้อนของ "ของฟรี" (สมการที่ควรคิดก่อนเลือก)

จากการรื้อระบบ "ของฟรีที่ไปต่อไม่ได้" ให้ SME ไทยหลายสิบเคสในปี 2025–2026 ต้นทุนแฝงแยกได้ 4 ก้อนเสมอ:

**ก้อนที่ 1: ค่าข้อความ/โควตาของช่องทาง** — LINE OA แผนฟรี broadcast ได้ประมาณ 300 ข้อความ/เดือน (ตัวเลข ณ มิ.ย. 2026 — เช็กหน้า pricing ทางการก่อนเสมอ) ขณะที่แผนเสียเงินเริ่มราว 1,200–1,280 บาท/เดือน ได้ 5,000–15,000 ข้อความ และข้อความส่วนเกินคิดราว 0.06–0.10 บาท/ข้อความ. จุดที่คนพลาดบ่อย: การ "ตอบแชต 1:1 เมื่อลูกค้าทักก่อน" ไม่กินโควตา broadcast — แต่แคมเปญ/การตามลูกค้า (push) กิน. ร้านที่ followers โตเร็วจะชนเพดานฟรีภายใน 1–2 เดือน.

**ก้อนที่ 2: ค่า token LLM** — ถ้าทำ AI จริง (ไม่ใช่สคริปต์) ต้องเรียก API ของ Claude/GPT/Gemini ซึ่งคิดตาม token. **ภาษาไทยใช้ token มากกว่าอังกฤษ 2–3 เท่า** ทำให้ค่าใช้จ่ายจริงสูงกว่าที่เห็นใน pricing page ฝรั่ง — รายละเอียดอยู่ใน [ต้นทุน token ต่อข้อความจริงของ chatbot ไทย](/blog/ai-chatbot-ต้นทุน-token-ต่อข้อความ-sme-2026). Free tier ของ LLM API (เช่น Gemini ระดับฟรี) มี rate limit ต่ำและเงื่อนไขการใช้ข้อมูลที่ต้องอ่านดีๆ ก่อนใช้กับแชตลูกค้าจริง.

**ก้อนที่ 3: ค่าเวลาของตัวเอง** — ติดตั้งครั้งแรก 10–20 ชั่วโมง (เลือก tool, ต่อ LINE/Facebook, เขียน FAQ, ทดสอบ) บวกดูแลต่อเนื่อง 2–5 ชั่วโมง/สัปดาห์ (อัปเดตคำตอบ, แก้เคสที่บอตตอบพลาด, จัดการ webhook หลุด). คิดค่าแรงตัวเองชั่วโมงละ 300–500 บาท เท่ากับ "ของฟรี" มีต้นทุนเดือนแรก 5,000–10,000 บาทโดยไม่รู้ตัว — แพงกว่าแพ็กเกจเริ่มต้นของหลายเจ้า.

**ก้อนที่ 4: ค่าความเสี่ยง** — ของฟรีแทบไม่มี guardrail: ไม่มีระบบกันบอตมั่วราคา, ไม่มี logic ส่งต่อคนเมื่อไม่แน่ใจ, ไม่มี log สำหรับ PDPA. บอตที่ตอบราคาผิดให้ลูกค้า 1 ครั้งอาจเสียหายมากกว่าค่าระบบทั้งปี — เรื่องนี้เขียนละเอียดใน [วิธีป้องกัน chatbot หลอน](/blog/ai-chatbot-หลอน-hallucination-ป้องกัน-sme-ไทย-2026).

> **สมการต้นทุนจริง:** ต้นทุนของฟรี = ค่าโควตาเกิน + ค่า token + (ชั่วโมงที่ใช้ × ค่าแรง/ชม.) + ความเสี่ยงตอบผิด. ถ้าผลรวมเกินค่าแพ็กเกจรายเดือนของระบบที่มีคนดูแลให้ — ของฟรีคือตัวเลือกที่แพงกว่า.

## ของฟรี "เอาอยู่" เมื่อไหร่ — 4 เคสที่เราแนะนำให้ใช้ฟรีไปก่อน

ไม่ใช่ทุกธุรกิจต้องจ่าย. เราบอกลูกค้าตรงๆ ว่า **อย่าเพิ่งจ้างใคร** ถ้าตรงตามนี้:

1. **ลูกค้าทักน้อยกว่า ~100 แชต/เดือน** — ปริมาณนี้ตอบเองไหว หรือ auto-reply คีย์เวิร์ดก็ครอบคลุม 70–80% แล้ว
2. **คำถาม 10 อันดับแรกซ้ำกันชัดเจน** — เวลาเปิด/ราคา/ที่อยู่/วิธีสั่ง → ตั้งคีย์เวิร์ดฟรีของ LINE OA ครอบได้เกือบหมด ไม่ต้องใช้ LLM เลย
3. **ยังไม่มีระบบหลังบ้านให้เชื่อม** — ถ้าไม่มีระบบจอง/สต็อก/CRM ที่บอตต้องไปดึงข้อมูล ข้อได้เปรียบของระบบเสียเงินจะยังไม่ออกฤทธิ์
4. **อยากเรียนรู้ก่อนลงทุน** — ลองของฟรี 1–2 เดือนเพื่อเก็บ "คำถามจริงของลูกค้า" เป็นข้อมูลตั้งต้น — ชุดคำถามนี้มีค่ามากเวลาทำระบบจริงทีหลัง

เริ่มจากศูนย์แบบไม่เขียนโค้ด ดูคู่มือ [DIY chatbot ไม่ต้องเขียนโค้ด](/blog/diy-chatbot-sme-ไม่ต้องเขียนโค้ด) ก่อนได้เลย.

## 7 สัญญาณว่าธุรกิจ "โตเกินของฟรี" แล้ว

1. **ชนโควตาข้อความ 2 เดือนติด** — ค่าส่วนเกิน + เวลาที่เสียเริ่มแพงกว่าแพ็กเกจจริง
2. **แชตเกิน ~100–150 เรื่อง/เดือน และครึ่งหนึ่งมาตอนปิดร้าน** — ลูกค้าเริ่มหายไปหาคู่แข่งที่ตอบเร็วกว่า
3. **ต้องเช็กของ/เช็กคิวก่อนตอบทุกครั้ง** — สคริปต์ฟรีทำไม่ได้ ต้องมี integration กับระบบจริง ([ตัวอย่างการเชื่อม POS/CRM/สต็อก](/blog/ai-chatbot-เชื่อมระบบหลังบ้าน-pos-crm-สต็อก-sme-2026))
4. **สินค้า/บริการราคาสูง** — ความเสี่ยงบอตมั่วราคา 1 ครั้ง แพงกว่าค่าระบบหลายเดือน ต้องมี guardrail จริง
5. **ทีมตอบแชตเกิน 1 คน** — เริ่มต้องการ handoff, การแบ่งเคส, log ตรวจย้อนหลัง
6. **ใช้เวลาดูแลบอตเกิน 5 ชม./สัปดาห์** — เอาเวลาไปขายของได้มากกว่า
7. **เริ่มกังวลเรื่อง PDPA** — เก็บข้อมูลลูกค้าในแชตโดยไม่มี consent/log ที่ถูกต้อง มีโทษปรับสูงสุดหลักล้าน ([คู่มือ PDPA สำหรับ chatbot](/blog/pdpa-ai-chatbot-sme-ไทย-2026))

นับได้ 3 ข้อขึ้นไป = ถึงเวลาคิดเรื่องระบบจริง. งบเท่าไหร่ถึงพอ ดูตัวเลขทุก tier ที่ [AI Chatbot ราคาเท่าไหร่ 2026 — คู่มือคำนวณงบ SME](/blog/ai-chatbot-ราคา-2026-คู่มือ).

## โตเกินฟรีแล้วไปไหนต่อ — 3 เส้นทาง + งบจริง

**เส้นทาง 1: จ่ายแพลตฟอร์มสำเร็จรูป (เริ่ม ~590–2,000 บาท/เดือน)** — อัปเกรด free tier เดิมเป็นแผนเสียเงิน. เร็วสุด แต่ฟีเจอร์ AI จริงมักอยู่แผนแพง และย้ายค่ายทีหลังเจ็บตัว (ประวัติแชต/contact ติดอยู่กับเขา).

**เส้นทาง 2: DIY แบบจริงจัง (VPS + n8n/Dify + LLM API, ~500–2,500 บาท/เดือน)** — คุมทุกอย่างเอง ไม่มี lock-in. เหมาะเมื่อมีคน technical ในทีม. เริ่มจาก [คู่มือ n8n สำหรับ SME ไทย](/blog/n8n-สำหรับ-sme-ไทย-คู่มือเริ่มต้น).

**เส้นทาง 3: จ้างทีมทำ + ดูแลให้ (setup หลักหมื่น + รายเดือนหลักพัน)** — ได้ guardrail, การทดสอบก่อนเปิดใช้, integration หลังบ้าน, และคนรับผิดชอบเมื่อพัง. คุ้มเมื่อแชตเยอะหรือความเสี่ยงตอบผิดสูง. วิธีเลือกไม่ให้โดนหลอก อ่าน [AI Agency ไทย เลือกยังไง 2026](/blog/ai-agency-ไทย-เลือกยังไง-2026).

## FAQ — คำถามที่ถูกถามบ่อยเรื่อง AI chatbot ฟรี

### Q1: AI chatbot ฟรีที่ดีที่สุดปี 2026 คือตัวไหน?
ไม่มีคำตอบเดียว — ถ้าลูกค้าอยู่บน LINE ให้เริ่มที่ auto-reply ของ LINE OA แผนฟรี (0 บาท, ตั้งเสร็จใน 1 ชม.). ถ้าต้องการ AI เข้าใจภาษาจริงและมีคน technical ให้ self-host Dify หรือ n8n (ฟรี license แต่จ่าย VPS + token). ถ้าไม่มีคน technical และอยากลองบอตไทย ให้ลอง free tier ของ ZWIZ/Botnoi ก่อน.

### Q2: LINE OA แผนฟรีทำ AI chatbot ได้ไหม?
ได้บางส่วน — แผนฟรีให้ auto-reply ตามคีย์เวิร์ดซึ่งไม่ใช่ AI. ถ้าอยากได้ AI จริงต้องต่อ webhook ไปหา LLM ภายนอก (ผ่าน Dify/n8n/โค้ดเอง) ซึ่งตัว LINE ไม่เก็บเงินเพิ่มสำหรับการตอบแชต 1:1 แต่เราจ่ายค่า token LLM เอง — ดูคู่มือเต็มที่ [AI Chatbot บน LINE OA สำหรับ SME](/blog/ai-chatbot-line-oa-สำหรับ-sme-2026-คู่มือเต็ม).

### Q3: ต้นทุนแฝงก้อนไหน "แพงที่สุด" ในทางปฏิบัติ?
ก้อนเวลา. ค่าโควตากับ token รวมกันมักไม่เกินหลักร้อยถึงพันต้นๆ ต่อเดือนสำหรับร้านเล็ก แต่เวลา 10–20 ชม.ติดตั้ง + 2–5 ชม./สัปดาห์ที่หายไปจากการขายของจริง คือก้อนที่เจ้าของร้านประเมินต่ำที่สุดและเจ็บที่สุด.

### Q4: Self-host (Dify/Flowise/n8n) ฟรีจริงไหม?
License ฟรีจริง แต่ระบบรันจริงมีค่า VPS ~250–500 บาท/เดือน, ค่า token ตามการใช้, และ "ภาษีดูแลระบบ" — อัปเดตเวอร์ชัน, backup, ความปลอดภัย. ถ้าไม่มีใครในทีมเคยดูแลเซิร์ฟเวอร์ อย่าเลือกเส้นนี้เป็นทางแรก.

### Q5: ใช้ free tier กับข้อมูลลูกค้าจริง ผิด PDPA ไหม?
ไม่ผิดโดยอัตโนมัติ แต่ต้องตรวจ 3 อย่าง: (1) ผู้ให้บริการเก็บข้อมูลแชตไว้ที่ไหน/นานเท่าไร, (2) เงื่อนไข free tier บางเจ้าใช้ข้อมูลเราเทรนระบบได้ — ต้องอ่าน ToS, (3) เรามีช่องทางแจ้ง/ขอ consent ลูกค้าหรือยัง. แผนฟรีมักไม่มีเครื่องมือช่วยเรื่องนี้เลย ซึ่งเป็นความเสี่ยงเงียบที่เจ้าของธุรกิจรับเอง.

### Q6: เมื่อไหร่ควรเลิกใช้ของฟรี?
เมื่อนับสัญญาณจากหัวข้อ "7 สัญญาณ" ได้ 3 ข้อขึ้นไป หรือเมื่อคำนวณสมการต้นทุนจริง (โควตา + token + เวลา × ค่าแรง + ความเสี่ยง) แล้วเกินค่าแพ็กเกจของระบบที่มีคนทำให้ — จุดนั้นของฟรีกลายเป็นตัวเลือกที่แพงกว่าแบบเงียบๆ แล้ว.
